CLAP Data

Data Display

Data from the CLAP is shipped directly to the host computer during operation and stored on the host hard drive (if the host is not operational, then no data from the CLAP will be saved). If data is being captured, then the string produced by the CLAP will be visible on the user interface under the General Tab in the box labeled "CLAP" in the textbox "Data Stream". If the server is on but there is no power, this box will display the text "TURN ON POWER"; if the server is not present or operational, then this box will contain no text.

Data Storage

The data is stored in the same directory as the CRDS/PAS data, C:\CRDS_PAS Data, in a sub-directory entitled CLAP. A new file is opened each time the user interface is opened. If a directory with the current date is not present, one will be created. Each file for the day is stored as text and uses the naming convention yyyymmdd_hhmmss.txt, where the date and time correspond to the desktop date and time that the file was opened. Each data file will contain a comma delimited string of text with the first entry corresponding to the time of the file entry and the remaining data being the string displayed in the textbox "Data Stream". The date/time format is yyyy-mm-ddThh:mm:ssZ, where the time is in UTC.

Sending Data for Processing

Once data is collected, the data must be shipped to John Ogren and his group. The data is shipped via anonymous ftp to the address

ftp://ftp.cmdl.noaa.gov/incoming/aer/air
If necessary, the file may be compressed using zip or bzip2.
